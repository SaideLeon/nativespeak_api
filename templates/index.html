Resumo

Este documentário descreve o propósito do projeto NativeSpeak, a API Django que suporta o front-end demo. O objetivo é ter um backend leve que substitua o uso do localStorage do front-end por um armazenamento persistente, com endpoints REST para configurações, ferramentas, logs e mais.

Arquitetura

Visão geral

O projeto é uma API construída com Django + Django REST Framework, organizada em um app principal chamado core. Os modelos centrais incluem:

  • Setting: chaves e valores JSON para configurações da aplicação.
  • Tool: ferramentas configuráveis que o front-end pode listar e usar.
  • LogEntry: entradas de log estruturadas.
  • Achievement: conquistas do usuário.
  • Notification: notificações que podem ser criadas pelo backend.
  • Presence: presença online de usuários.
  • TodoItem: itens de tarefa simples.
  • Lesson: lições com slug e conteúdo JSON.
  • UIState: estados da interface armazenados por chave.

Fluxo de dados

O front-end realiza requisições para os endpoints REST expostos por viewsets e viewsets customizados. As operações CRUD são mapeadas diretamente para modelos Django e serializadores DRF.

Decisões técnicas

Por que Django + DRF?

Django fornece um ORM maduro, sistema de administração e convenções que aceleram o desenvolvimento. Django REST Framework abstrai rapidamente a exposição de APIs REST com autenticação, permissões e serialização.

Admin moderno com Unfold

O projeto está integrado ao tema django-unfold, um tema moderno para o admin Django baseado em Tailwind. Isso melhora a usabilidade e permite uso de componentes prontos. O settings.py foi atualizado para ativar o Unfold e fornecer algumas opções iniciais.

Como rodar

Passos curtos (PowerShell):

py -3 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt  # ou pip install django djangorestframework django-cors-headers django-unfold django-crispy-forms
py -3 manage.py migrate
py -3 manage.py runserver

Trechos de código

Model `Setting`

from django.db import models

class Setting(models.Model):
    key = models.CharField(max_length=200, unique=True)
    value = models.JSONField()

    def __str__(self):
        return self.key

Exemplo de chamada ao endpoint de settings (fetch)

fetch('/api/settings/')
  .then(r => r.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

Personalização do Admin

O admin já registra todos os modelos na app core usando classes baseadas em Unfold `ModelAdmin`. Você pode personalizar cada admin para ajustar:

  • Campos mostrados na listagem (list_display).
  • Filtros (list_filter).
  • Campos pesquisáveis (search_fields).

Roadmap de melhorias

  1. Adicionar Tailwind build pipeline (para personalização avançada) usando Node/Tailwind ou `django-tailwind`.
  2. Adicionar widgets Unfold/Alpine.js para experiências interativas no admin.
  3. Adicionar testes automatizados e CI para garantir estabilidade.
